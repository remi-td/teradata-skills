# Skill name

**Chosen name: `teradata-react`**

## Candidates considered

| Name              | Pro                                      | Con                                       |
|-------------------|------------------------------------------|-------------------------------------------|
| `teradata-react`  | Names the two load-bearing technologies  | Hints React only; backend is FastAPI      |
| `teradata-webapp` | Tech-agnostic                            | Vague; doesn't trigger on "React" prompts |
| `vantage-app`     | Marketing-aligned                        | Ambiguous (Vantage is many things)        |
| `teradata-dataapp`| Communicates intent                      | "dataapp" is non-standard                 |
| `teradata-stack`  | Suggests opinionated bundle              | Doesn't say *what* stack                  |

## Why `teradata-react`

- **Discoverability**: agents and humans searching "react" or "teradata"
  both hit it.
- **Honest scope**: the differentiated part *is* the React frontend
  pattern; FastAPI is the standard pairing.
- **Consistent** with sibling skills (`teradata-sql-analytics`,
  `teradata-query`, `teradata-brand`).

The SKILL.md description makes the full-stack scope explicit so the
trigger isn't misread as "React tips only".
